Introduction

Purpose

The tru package uses a trust-region method to find a (local) unconstrained minimizer of a differentiable objective function of many variables . The method offers the choice of direct and iterative solution of the key trust-region subproblems, and is most suitable for large problems. First derivatives are required, and if second derivatives can be calculated, they will be exploited—if the product of second derivatives with a vector may be found, but not the derivatives themselves, that may also be exploited.

Method

A trust-region method is used. In this, an improvement to a current estimate of the required minimizer, is sought by computing a step . The step is chosen to approximately minimize a model of within a trust region for some specified positive "radius" . The quality of the resulting step is assessed by computing the "ratio" . The step is deemed to have succeeded if the ratio exceeds a given , and in this case . Otherwise , and the radius is reduced by powers of a given reduction factor until it is smaller than . If the ratio is larger than , the radius will be increased so that it exceeds by a given increase factor. The method will terminate as soon as is smaller than a specified value.

Either linear or quadratic models may be used. The former will be taken as the first two terms of a Taylor series about , while the latter uses an approximation to the first three terms , for which is a symmetric approximation to the Hessian ; possible approximations include the true Hessian, limited-memory secant and sparsity approximations and a scaled identity matrix. Normally a two-norm trust region will be used, but this may change if preconditioning is employed.

An approximate minimizer of the model within the trust region is found using either a direct approach involving factorization or an iterative (conjugate-gradient/Lanczos) approach based on approximations to the required solution from a so-called Krlov subspace. The direct approach is based on the knowledge that the required solution satisfies the linear system of equations involving a scalar Lagrange multiplier . This multiplier is found by uni-variate root finding, using a safeguarded Newton-like process, by the GALAHAD packages TRS or DPS (depending on the norm chosen). The iterative approach uses the GALAHAD package GLTR, and is best accelerated by preconditioning with good approximations to using GALAHAD's PSLS. The iterative approach has the advantage that only matrix-vector products are required, and thus is not required explicitly. However when factorizations of are possible, the direct approach is often more efficient.

Call order

To solve a given problem, functions from the tru package must be called in the following order:

  • tru_initialize - provide default control parameters and set up initial data structures
  • tru_read_specfile (optional) - override control values by reading replacement values from a file
  • tru_import - set up problem data structures and fixed values
  • solve the problem by calling one of
    • tru_solve_with_mat - solve using function calls to evaluate function, gradient and Hessian values
    • tru_solve_without_mat - solve using function calls to evaluate function and gradient values and Hessian-vector products
    • tru_solve_reverse_with_mat - solve returning to the calling program to obtain function, gradient and Hessian values, or
    • tru_solve_reverse_without_mat - solve returning to the calling prorgram to obtain function and gradient values and Hessian-vector products
  • tru_information (optional) - recover information about the solution and solution process
  • tru_terminate - deallocate data structures

See the examples tab for illustrations of use.
